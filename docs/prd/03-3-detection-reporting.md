# 🧱 기능 요구사항 - 탐지·보고·점검 모듈

## 4. 탐지 모듈

- STR / CTR / WLF 실행 및 운영 탐지 (실데이터 적용, 스냅샷 고정)

## 5. 보고 모듈

- STR / CTR 자동 보고서, FIU 템플릿 기반 생성·검증

## 6. 점검 수행 모듈

- STR / CTR / RBA / 요주의인물 점검(배치 Instance/Execution 구조)

### 핵심 플로우(요약)

**데이터 유입 → 탐지 → 케이스 생성 → 조사·증빙 → 결재 → 보고(전송/수신확인) → 감사/리포팅**

**정책/기준**: 초안→리뷰→승인→배포(스냅샷)→탐지엔진 반영
